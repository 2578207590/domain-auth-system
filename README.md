# 🛡️ 域名授权系统

> 基于 **PHP + MySQL** 的轻量级域名授权管理系统。支持卡密生成与激活、域名续费、泛域名授权、客户端自动验证。

---

## 📋 功能总览

| 模块 | 功能 |
|------|------|
| 🎫 卡密管理 | 生成、导入、导出卡密；支持批量删除、批量修改有效期 |
| 🌐 域名管理 | 添加、修改、删除域名；支持续费、封禁/解封、泛域名授权 |
| 🔐 客户验证 | 网页前端 JS 自动检测域名授权状态，未授权/已到期弹窗激活 |
| ⚙️ 系统设置 | 修改管理员账号密码（bcrypt 加密存储） |
| 📋 操作日志 | 记录所有操作，支持搜索、清理 |
| 📊 统计面板 | 授权域名数量、到期域名、封禁域名、卡密使用率 |

## 🎫 卡密管理

### 生成卡密

支持设置**自定义授权天数**：

- 系统默认（365 天）
- 30 / 90 / 180 / 365 天预设
- **永久有效**（expire_days = 0）
- **自定义天数**（任意数字，如 720 天）

### 批量操作

- 批量删除已选卡密
- 批量修改有效期（支持自定义天数）
- 清空所有已使用卡密

### 导出卡密

- 纯码导出（一行一个卡密码）
- 完整导出（含域名、使用时间）
- 按状态筛选导出（未使用 / 已使用）

### 状态与使用

卡密分为：**未使用 → 已使用**。使用后自动记录绑定域名和激活时间。

## 🌐 域名管理

### 域名授权

- 添加域名时选择授权天数（或永久）
- 支持**泛域名**（如 `*.example.com`），需管理员手动添加
- 已授权域名到期前 30 天自动显示 **"即将到期"** 橙色提示

### 续费

- 后台一键续费：+30 / +90 / +365 天或设为永久
- 续费时**叠加到期时间**（如原来剩 60 天 + 续 90 天 = 150 天）
- 已到期域名续费后恢复正常状态

### 修改域名

修改域名时可同时修改到期时间：
- 保持原到期
- 设为永久
- 立即到期
- 重置为 N 天（支持自定义天数）

### 状态显示

| 状态 | 徽章 | 说明 |
|------|------|------|
| ✅ 正常 | 绿色 | 授权有效期内 |
| ⚠️ 即将到期 | 橙色 | 剩余 < 30 天 |
| ⏰ 已到期 | 红色 | 已过期，需续费 |
| 🚫 封禁 | 红色 | 管理手动封禁 |

## 🔐 客户端验证

部署 `auth.js` / `auth_en.js` 到需要授权保护的网页中：

- 页面加载时自动检测当前域名授权状态
- **未授权** → 弹出遮罩层，显示激活界面
- **已到期** → 弹出遮罩层，显示续费界面
- **已封禁** → 弹出遮罩层，禁止访问
- **授权正常** → 无动作，页面正常显示

### 激活流程

1. 用户输入卡密
2. 点击激活按钮（含 loading 动画）
3. 成功后自动刷新页面
4. 显示真实到期时间（不硬编码 365 天）

### 泛域名匹配

客户端 JS 支持泛域名匹配（如 `*.example.com` 匹配 `sub.example.com`），匹配包含点边界检查，防止误匹配。

## ⚙️ 系统管理

### 安全特性

- **CSRF 保护** — 所有 POST 操作均使用随机 Token 验证
- **会话超时** — 后台 30 分钟无操作自动退出
- **登录限制** — 5 次错误密码后冻结 15 分钟
- **密码加密** — 使用 bcrypt 加密存储
- **XSS 防护** — 所有输出经过 htmlspecialchars 转义
- **事务保护** — 卡密激活操作使用数据库事务，防止数据不一致

## 🚀 安装部署

### 环境要求

- **PHP** >= 7.0（推荐 7.4+）
- **MySQL** 5.6+ / MariaDB
- 服务器需支持 `mysqli` 扩展和 `password_bcrypt`

### 安装步骤

```bash
# 1. 上传所有文件到网站根目录
# 2. 访问安装向导
```

访问 `http://your-domain.com/install.php`

填写：

| 字段 | 说明 |
|------|------|
| 数据库地址 | 通常为 localhost |
| 数据库用户名 | MySQL 用户 |
| 数据库密码 | MySQL 密码 |
| 数据库名 | 需提前创建或使用已有库 |
| 管理员账号 | 后台登录用 |
| 管理员密码 | 后台登录用（bcrypt 加密） |

安装完成后自动跳转到 `admin.php` 后台。

### 配置客户端验证

1. 修改 `auth.js` 和 `auth_en.js` 中的 API 地址：

```javascript
const API = "https://your-domain.com/ym/api.php";
```

2. 将 `auth.js` / `auth_en.js` 部署到需要保护的页面中：

```html
<script src="/path/to/auth.js"></script>
```

## 🔌 API 接口

### 检查域名授权状态

```
GET api.php?act=check&domain=xxx
```

```json
{"code": 1, "expire_time": "2027-05-06 12:00:00"}   // 授权正常
{"code": 2, "msg": "授权已到期，请续费"}                // 已到期
{"code": -1, "msg": "该域名已被封禁"}                   // 被封禁
{"code": 0}                                             // 未授权
```

### 使用卡密激活域名

```
GET api.php?act=active&domain=xxx&code=CARD123
```

```json
{"code": 1, "msg": "激活成功", "expire_time": "2027-05-06"}
{"code": 0, "msg": "卡密无效或已使用"}
```

### 获取授权列表

```
GET api.php?act=list
```

返回所有有效授权域名数组（供前端泛域名兜底匹配）。
